# Godot GDScript 中文注释

按下面规则写注释：

- 只给 Godot 的 `.gd` 文件写注释。
- 注释一律使用中文。
- 语言保持精炼、准确、直接，不写空话。
- 注释放在对应语句的正上方，不写在行尾。
- 重点给关键逻辑加注释，不要把显而易见的赋值逐行重复一遍。

优先注释这些位置：

- 重要函数。每个重要函数上方都要说明这个函数是做什么的。
- 生命周期函数、状态切换函数、事件回调、工具函数。
- 关键 `@export var`、状态变量、`@onready var`、`signal`。
- 带副作用的逻辑，例如生成目标点、修改状态、延迟调用、销毁节点、移动、攻击、受伤、导航、交互。
- 不直观的判断分支、提前返回、边界处理。

写注释时遵守这些要求：

- 用一句短句把意思说清楚。
- 优先解释为什么要这样做，或这段逻辑的作用。
- 避免只把代码换一种说法重复一遍。
- 注释要贴近当前脚本语境，术语要和文件里的命名一致。
- 如果已有注释含糊、啰嗦或不准确，直接改成更准确的中文。

不要注释这些位置：

- 一眼就能看懂的简单赋值。
- 没有信息增量的转发代码。
- 会干扰阅读的大段解释。

推荐风格：

```gdscript
# 进入巡逻状态时重置本轮计数。
func _on_enter() -> void:
	current_patrol_index = 0

# 导航结束后重新挑选一个随机目标点。
if navigation_agent_2d.is_navigation_finished():
	set_movement_target()

# 只在目标无效时提前返回，避免重复寻路。
if !target_node:
	return
```
